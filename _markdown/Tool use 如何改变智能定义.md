Turing 1950 年那篇《Computing Machinery and Intelligence》定下了关于机器智能的第一个隐含假设：智能是一种发生在封闭系统里的属性。一台机器关在房间里，只通过文字和外界交换信息，如果它的回答让人分辨不出是不是人，它就算有智能。这个设定很美，它把问题干净化了，但它也悄悄藏了一个前提——**智能是不需要外部工具的**。一台足够聪明的机器应该能在自己的脑子里把事情想明白。

过去七十年，这个前提塑造了一整条技术路线，也塑造了我们评估进步的方式。从 ELIZA 到 GPT-3，衡量一个系统聪不聪明，主要看它在一段封闭对话里、不接入任何外部资源的情况下能答出什么。

2022 到 2025 这几年，这个前提被一步一步拆掉了。到 2025 年末，它基本已经不成立了。一个今天还在用"模型在封闭对话里能答什么"来衡量 AI 进展的人，大概率已经落后了两代。

## 一、Oracle 时代

GPT-3 在 2020 年夏天放出来的时候，主导的使用姿态是把它当作一个**神谕**。你给它一段 prompt，它给你一段续写。中间没有工具、没有状态、没有动作。所有智能都必须被塞进两个 token 之间那次 forward pass 里。

这个时代的惊艳来自于"居然能"——它居然能写像样的诗，居然能翻译，居然能解一些数学题，居然能续写一段 Python。所有的努力都集中在怎么把更多的知识和推理能力压进模型参数，怎么让单次 forward pass 产生更多可用的东西。prompt engineering 在这个阶段成了一门手艺，本质上是在一个封闭神谕面前学习怎么提问。

这个范式有一个清晰的上限。模型知道的东西局限于它训练时看过的东西。它算数算不准，因为它用语言模型的方式在逼近算术。它访问不了今天的新闻，数据库，API，文件系统。它没办法执行代码验证自己的输出。所有这些限制在当时看来是"模型还不够强"的问题。

直到有人意识到这里要解的根本不是"把模型做得更强"那种问题——它是一个**姿势问题**。

## 二、ReAct 和 function calling

2022 年 10 月，Yao 等人在一篇叫《ReAct: Synergizing Reasoning and Acting in Language Models》的论文里提出了一个朴素但关键的想法：让模型在推理的时候可以穿插"动作"——比如去搜索引擎查一下，去一个 wiki API 里拉一段内容——再把返回的结果带回到推理里。这件事看起来像一个工程 trick，但它背后的意思很深：**模型的认知过程可以突破自己的参数边界**。思考不再局限在模型脑子里，它可以把一部分工作外包给世界。

几个月之内，这个想法就从论文变成了产品。2023 年 3 月 OpenAI 给 GPT-4 加上了 plugin 接口，6 月推出 function calling，让开发者能用结构化的方式把工具喂给模型。与此同时开源社区里 LangChain 和 AutoGPT 开始流行，它们把"模型 + 工具 + 循环"这个模式包装成可复用的框架。这段时间整个 AI 工程文化的中心，从"怎么写好 prompt"慢慢挪到了"怎么给模型配好工具"。

但 2023 年的 tool use 还相当脆。模型调用工具的成功率不高，在长对话里容易丢上下文，经常跑偏。AutoGPT 最有名的特征是它能跑起来——但它几乎跑不完任何一件真实的事，它会在一个无限循环里烧掉你账户里的钱然后卡住。那一年让人激动的东西叫**可能性**，可用性还远没到。

## 三、2024：从玩具到工具

2024 年是关键的过渡年。几件事在这一年汇聚了。

模型在 tool use 这件事上从勉强可用变成了可靠。Claude 3 系列在 2024 年 3 月放出来之后，很多开发者第一次感到模型能"听话地"使用一组工具——它会等返回结果再往下走，不会在中间幻觉一个假的 API response。GPT-4 Turbo 和 Gemini 1.5 也在同时期把这条线推到了类似的可用水位。

同一年，模型开始能用**计算机本身**作为工具。Anthropic 在 2024 年 10 月发布了 Computer Use，让 Claude 可以直接操控一个桌面——看屏幕、动鼠标、打字。同一年早些时候 Cognition 用一个叫 Devin 的 demo 把"AI 软件工程师"这个概念扔到了公众面前，引起了巨大争议，但也改变了大家对"AI 能做到什么"的心理预期。无论 Devin 那个 demo 的含水量怎么估算，它把 agent 这个词从 AutoGPT 式的玩具领域，拉进了"也许真能用来干活"的讨论。

与此同时，benchmark 的重心悄悄挪了位。2023 年大家还在比 MMLU、GSM8K 这些静态知识和推理题；2024 年开始，真正被严肃看待的是 SWE-bench（让模型独立解决真实 GitHub issue）、WebArena（让模型在真实网页上完成多步任务）、τ-bench（客服场景下的完整任务）。这些 benchmark 衡量的对象换了，从"模型能答什么"挪到了"agent 能完成什么"。底层的评价范式换了。

## 四、2025：agent 爆发

2025 年到来之后，这件事突然就不再是"新东西"了。

它变成了**默认工作方式**。一个 2025 年中写代码的工程师，坐在编辑器前的姿势和三年前完全不一样。三年前他自己打字，偶尔问 AI 一两句。现在他开着一个或几个 agent 会话让它们独立跑任务，自己在几个任务之间切换做 review。一个产品团队讨论"怎么做 X"，默认的第一步是问"能不能让 agent 先跑一版出来看看"。运营、分析、客服、法务，每个职能里都有人在尝试把重复工作整块交给 agent 处理。

发生变化的有几层。

**时间尺度**变了。以前"模型帮我干一件事"的单位是秒到分钟。2025 年开始，"让它干几个小时"成了正常操作。一个 coding agent 接到一个复杂的重构任务，它可能会跑四十分钟，读几十个文件，写十几次尝试，其中一半被自己的测试推翻重来。这个过程中没有人类 in-the-loop。人类只在开头定义问题、在结尾验收结果。

**并行度**变了。因为单位任务时间变长，也因为边际成本便宜，一个用户同时跑多个 agent 变成了普通操作。你在 A 任务上等着，就去启动 B 任务，B 任务在跑的时候你回来看 A 的进度。人类的注意力被重新安排成了"agent 调度员"的角色，执行者这个身份交了出去。

**协作模式**变了。2025 年很多严肃的工作流里，在跑的已经是一组分工明确的 agent，单 agent 反倒成了简单场景的特例：一个负责规划，一个负责执行，一个负责审查，一个负责回放日志找 bug。多 agent 系统从论文话题变成了工程常态。与此同时，MCP（Model Context Protocol）这类开放协议把工具接入从每家厂商自己造轮子变成了可以跨生态复用的标准——这是一件从外部看没什么声响、但从内部看决定性的事。

**基准**也基本失效了一次。2025 年上半年多个模型的 SWE-bench 分数冲到了 70% 以上，在两年前的人看来这是科幻片里的数字。所以大家又忙着设计更难的 benchmark，比如 SWE-bench Verified、SWE-Lancer、让 agent 跑真实的 freelance 任务，看最后有多少交付物让人类愿意付款。Benchmark 自己在追着 agent 的能力跑。

回过头看，从 ReAct 那篇论文到 2025 的爆发，一共走了大约三年。中间没有任何一次算力意义上的"质变"——模型架构还是 transformer，scaling law 还是那个 scaling law。真正变的是**姿势**：从让模型在脑子里想明白，转变为让模型调动外部世界的资源去把事情做成。

## 五、重新定义"智能"

这个过程改写了一个比产品设计深得多的东西：智能本身是什么。

Andy Clark 1998 年和 David Chalmers 合作写过一篇叫《The Extended Mind》的论文，主张人类的认知从来都不是发生在颅骨之内的。你用笔记本记电话号码，笔记本就是你记忆的一部分；你用计算器算账，计算器就是你算术能力的一部分；你给同事发消息问一个问题，对方的大脑就暂时并入了你的认知系统。Clark 的观点当时在哲学圈引起争议，因为它听起来像是放弃了"心智"这个概念——如果认知可以外溢到工具和他人身上，那大脑还有什么特殊性？

这个哲学问题，AI 用工程的方式回答了。

一个 2025 年的 agent，本质上是一个相对较小的核心模型 + 一组工具 + 一段记忆 + 可能若干其他 agent 组成的联合体。要评估它的智能，评估对象从核心模型的参数量，整体挪到了这个联合体的协同能力。把核心模型换掉一代，联合体继续工作；把工具换掉一组，同一个核心模型能做的事完全不同。**智能的载体已经从单个模型挪到了整个 agent 架构**。Clark 所说的"延展心智"，从一个哲学论证变成了工程默认。

这件事反过来也重新解释了人类智能。一个人的聪明程度，从来不只取决于他脑子里装了什么，同样取决于他身边放着哪些书、他认识哪些人、他能打给谁、他用什么工具在思考。我们过去把这些算作"外围"，是因为在人身上，核心和外围的界限比较稳定。AI 让核心这件事变成了可替换模块之后，界限本身也变得可见了。

## 六、什么是稳定的

模型会继续迭代。今天最强的模型到明年就会变成中等水平。如果一篇文章要写得能活过几代模型，它讨论的对象得挪开"某一代模型具体能做什么"这一层，去找那些在代际之间依然成立的结构。

关于 tool use 这件事，有几件事是稳定的。

**智能的计量单位永远回不到封闭系统了**。即使未来某一代模型的参数量大到它什么工具都不需要也能把很多事做完，人们也不会再回到只评估它闭卷成绩的状态。因为已经被证明，"开卷 + 工具 + 多 agent 协同"的架构在同等核心能力下总是更强。这是一个单向门，推开了就关不上。

**衡量 AI 系统的关键指标，从静态能力转到了动态编排**。一个系统的核心能力当然重要，但同等或更重要的，是它能调动多少工具、多准确地调动、在多长的任务里保持连贯、多少步之后还能自我纠正。这些属性在下一代模型里还在，再下一代模型里也还在，因为它们对应的是现实世界本身的结构——现实有很多工具，任务需要长时间执行，错误会累积需要被修正。

**agent 的设计经验开始独立于模型的代际积累**。怎么给 agent 设计好的工具接口，怎么管理它的上下文，怎么组织多 agent 之间的分工，怎么做可观测性和可回放，这些问题在 2025 年还相当年轻，但它们的答案在积累。每一代新模型来了之后，这些经验大部分迁移得过去。这是一种新的工程知识，独立于模型本身在变。

**关于人类认知的哲学视角，也不会退回去**。一旦我们在工程里承认了"延展心智"这种结构，再把人脑视为一个单点的封闭认知系统就显得陈旧了。这件事最终会渗透到教育、到组织设计、到我们对"什么算一个人的能力"这种基础判断里去。

---

Turing 那篇论文的标题是一个问题——机器能思考吗？七十五年之后，问题的形状已经改了。**机器能不能思考**这个问，预设了一个思考发生在单一主体里的封闭空间。真正被 2022 到 2025 这几年改写的，是这个预设本身。现在的问题是另一个：一个会调用工具、会和其他 agent 协作、会把任务拆成几十步并在出错时自我纠偏的系统，它的行为最终和"思考"这个词还有什么区别？

如果没有区别，那思考这件事就从来不是它过去被描述的那样。
