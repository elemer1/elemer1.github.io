## 起因

先说清楚：**我还没有孩子，甚至也还没结婚**。这是一次思想实验——

> *如果有一天我有了孩子，快到该学编程的年纪，我会怎么教 TA?*

给 TA 报班、买本书、买套编程玩具，当然都行。但会遇到一个问题——**编程这件事，对今天的孩子来说，看起来像魔法**。

TA 打开手机上的 App，屏幕亮起，有动画，有声音，有按钮。TA 问："这是怎么做出来的？" 大人一般回答："有程序员写了代码。" TA 再问："代码是什么？"

这时候多数大人会卡住。

因为"代码是什么"这个问题，要真讲清楚，你得从底下讲起：电怎么变成数字，数字怎么变成指令，指令怎么变成屏幕上的像素。而现代编程工具——Python、React、游戏引擎——把中间 20 层抽象全都封起来了。孩子能学会"拼图",但不知道图是从哪里来的。

所以这次思想实验的结论是：**与其让 TA 先学 Scratch，不如让 TA 先看到最底下那一层长什么样**。

于是——用 Claude Code 帮忙，一个下午，我用 C++ 写了一个能开机、能响应键盘、能跑命令的最小操作系统，一共 **492 行代码**。那个想象中的孩子还不知道哪一年会出现，但这个 OS 是实打实写完的。

这个项目叫 **mini-os**。

## 设计

### 选型

每一个决定背后都有一个问题：**"这对孩子理解'计算机是什么'有没有帮助？"**

| 决定 | 选项 A | 选项 B | 选 | 为什么 |
|------|--------|--------|-----|-------|
| 架构 | 32 位 i686 | 64 位 x86_64 | **i686** | 进长模式要 40 行汇编的 boilerplate，对孩子是噪声 |
| 引导 | GRUB + Multiboot1 | UEFI + Limine | **GRUB** | Multiboot 规范一页纸读完;UEFI 是天书 |
| 键盘 | 硬件中断 | 轮询 | **轮询** | 中断要先写 IDT/PIC 约 100 行，看不到实际效果;轮询 30 行，按下就有反应 |
| 语言 | C / C++ / Rust | — | **C++** | C 缺命名空间;Rust 的借用检查器对孩子是噪声;C++ 能用 namespace、constexpr、enum class 把意图写清楚，又完全贴硬件 |

原则只有一条：**砍掉所有不能对孩子解释的东西**。

### 架构

从按下电源到屏幕出现第一个字，全程能一步步讲完：

```
按下电源
   ↓
BIOS 找到光驱
   ↓
GRUB (装在 ISO 里) 被加载
   ↓
GRUB 读 grub.cfg → 找到 kernel.bin
   ↓
GRUB 根据 Multiboot1 头把内核装到物理内存 1 MiB
   ↓
跳到 _start (汇编)
   ↓
_start 设好栈指针 → call kmain
   ↓
kmain (C++) 初始化 VGA,打欢迎语
   ↓
shell::run() 开始轮询键盘,永不返回
```

**没有一步是"暂时别管"**。

### 代码组织

| 文件 | 行数 | 职责 |
|------|-----:|------|
| `src/boot.s`           | 36 | Multiboot1 头、`_start`、栈 |
| `linker.ld`            | 32 | 告诉链接器把内核放到 1 MiB |
| `src/kernel.cpp`       | 17 | `kmain()`:初始化 VGA、跳进 shell |
| `src/vga.{h,cpp}`      | 107 | 0xB8000 文字模式驱动 (颜色、滚屏、退格) |
| `src/io.h`             | 19 | `inb` / `outb` 端口 I/O 内联汇编 |
| `src/keyboard.{h,cpp}` | 80 | PS/2 扫描码 → ASCII，含 Shift / Caps |
| `src/shell.{h,cpp}`    | 135 | REPL:`help` / `echo` / `about` / `colors` / `clear` |
| `Makefile`             | 59 | 构建 + 打 ISO + 启 QEMU |
| `grub.cfg`             | 7  | GRUB 启动项 |
| **合计**               | **492** | |

## 哲学

### 1. 最小即可懂

**492 行能做一个真操作系统。**

很多东西看起来复杂，是因为它被做成了复杂——层层包装、无数抽象。但底下永远有一个最小可运行版本，而这个最小版本就是"懂"的起点。

对大人来说，这一点重要，因为我们太容易把复杂当成权威。对孩子来说更重要：**从一开始就让 TA 知道，真东西没那么多行**。

### 2. 无魔法：从 0xB8000 开始

打开 `src/vga.cpp`,能看到这一行：

```cpp
volatile uint16_t* const BUFFER = reinterpret_cast<uint16_t*>(0xB8000);
```

这一行说完了 VGA 文字模式的**全部秘密**:屏幕上每一个字，就是从内存地址 `0xB8000` 开始的一块内存里，一个 16 位数字——8 位 ASCII + 4 位前景色 + 4 位背景色。

不是"调用 `Print()`",不是"导入 Console 库",不是"创建 Window"——**就是写内存**。

孩子第一次听到这句话会震惊："屏幕是内存？"

对，屏幕是内存。你看到的每一个字符，都是某块内存里某个数字的样子。知道这一点以后，TA 再看任何 UI 框架，都会多一层透视。

### 3. 可见的因果

在 mini-os 里，从 TA 按下 `h` 到屏幕上出现 `h`,中间发生这些事：

```
键盘硬件 → PS/2 控制器端口 0x64 / 0x60
  → keyboard::poll() 读到扫描码 0x23
  → SC_LOWER[0x23] = 'h'
  → shell::run() 拿到 'h'
  → vga::putc('h')
  → 写到 0xB8000 + row*160 + col*2
  → 显卡扫描这块内存,在屏幕上画出 "h"
```

**每一步都落在仓库里的某一行代码**。能打开给 TA 看，能改，能弄坏重来。

这是现代应用编程学不到的东西。用 Python 写 GUI，从来不知道"按下 h"背后多少层。在 mini-os 里，**只有自己写的那些层，所以必然认识它们**。

### 4. C++ 的诚实

为什么不是 Python？不是 Go?

- Python **把内存藏起来了**——看不到指针，看不到地址。
- Go **藏起了垃圾回收的时机**——你不知道什么时候停顿。
- Rust **藏起了编译器为什么不过**——借用检查器对成年人都是难度。

**C++ 不藏**。指针是地址，类是一块内存，`reinterpret_cast` 就是"把这堆字节当那种类型看"。这种直白，对理解硬件是最贴近的。

C++ 的另一个好处：它允许你既写接近汇编的代码，又能用 namespace、`constexpr`、`enum class` 把意图表达清楚。`vga.cpp` 和 `shell.cpp` 读起来并不比 Python 差，**但它们直接操作硬件**。

## 教育意义

### 一起做，不是教

假如真有那么一天，我不打算"教" TA 写 OS——我打算**邀请 TA 一起改**。

- "这句欢迎语你想改成什么？"
- "`help` 里要不要加一个 `joke` 命令，每次随机打一个笑话？"
- "你想要什么颜色的提示符？"

改一行，`make run`,看结果。这是编程最甜美的反馈循环。TA 不是在"学编程",TA 是在**用编程让电脑做自己想让它做的事**——这两件事在孩子眼里是完全不同的。

### 每条命令后面都有故事

![help 命令](/assets/images/mini-os-shell-help.png)

运行 `make run` 以后，我打算这样演示：

**开机时**:"这是爸爸写的一个操作系统。整台电脑上除了它什么都没有——连 macOS 都没有。我们现在在另一个宇宙里。"

**输 `help`**:"这些命令的名字，都是爸爸在 `shell.cpp` 里写的一行 `if`。你想加一个新命令，就多写一个 `if`。"

**输 `echo hi from qemu`**:

![echo 命令](/assets/images/mini-os-shell-echo.png)

"你按下每一个字母，键盘先告诉操作系统一个数字 (叫扫描码),操作系统查表把数字换成字母，然后写到屏幕内存里。"

**输 `colors`**:看 16 色调色板。"屏幕每一个字后面有一个数字告诉显卡它是什么颜色。数字只有 0 到 15——所以只有 16 种颜色。"

### 什么年纪讲什么层

同一个仓库，随 TA 长大慢慢揭开。每一层都在等 TA。

| 年龄 | 能讲到的层 |
|------|-----------|
| 5-7 岁 | 只看启动画面和 `colors`。讲"屏幕上的字是'画'出来的"。 |
| 8-10 岁 | 讲命令，讲"程序就是 `if` 和 `for`"。让 TA 读一个 `if`。 |
| 11-13 岁 | 讲内存、地址、端口。打开 `vga.cpp` 解释 `0xB8000`。 |
| 14+ 岁 | 讲汇编、中断、保护模式。让 TA 读 `boot.s`。 |

### 最重要的教育意义

**让 TA 知道：这个世界不是被某家大公司做好了端给我们用的。**

"操作系统是大公司工程师写的"——这是 TA 未来会默认接受的信念。我希望 mini-os 给 TA 一个反例：**一个普通人，在一个下午，能写出一个能跑的操作系统**。

TA 长大不一定会成为程序员。可能更喜欢艺术、生物、做菜——那都很好。

但不管 TA 选什么，我希望这个小小的 OS 告诉 TA 一件事：

> **你看到的每一样东西——手机、游戏、网站、AI——都是人做的。是具体的、会累的、会犯错的、跟你我一样的人。**
>
> **所以如果你想做什么，不用问"我可以吗"。去做。**

## 运行它

```bash
cd ~/Projects/mini-os
make          # 编译内核,打 ISO
make run      # 启 QEMU,打开一扇窗
make clean    # 清除构建产物
```

可用命令：`help` / `echo <text>` / `about` / `colors` / `clear`。

## 下一步 (给 TA 长大后留的空间)

代码留了空隙等 TA:

- **时钟中断**:让光标闪烁。GDT + IDT + 8259 PIC 重映射。约 150 行。
- **贪吃蛇**:仍然用 VGA 文字模式画，WASD 控制。约 100 行。
- **多任务**:保存 / 恢复寄存器，两个任务交替跑。**这是 OS 开始变"真"的时刻**。
- **文件系统**:把 FAT12 读进来。
- **切到 64 位**:进长模式。这是和工业界真实 OS 拉齐的一步。

每一步都是一个新的周末项目。
