1993 年，网页只是带超链接的文档。Mosaic 还没学会把图片嵌进正文，JavaScript 尚未被发明，CSS 要到 1996 年才诞生。人们打开一个网页，看到的是纯文本和几个蓝色下划线——然后读。

三十多年过去，网页变重了。一个简单的博客首页动辄五兆、十几次请求、滚动时塞满动画。我反复问自己：多出来的那几兆，究竟给读者多了什么？

答案通常是：更少。

## 形式追随沉默

包豪斯说 form follows function。放到网页上，function 就是让你读完这句话。一切不为"读完"服务的东西都是噪音——渐显的大标题、悬浮的订阅弹窗、滚动视差、骨架屏、首屏大图、字体加载闪烁。每一个都在和内容争夺那几秒注意力。

所以我把它们全部砍掉了：

- 没有追踪脚本。
- 没有 Web Font，用你系统里已有的字体。
- CSS 内联不到 60 行，够了。
- 首页是一句自我介绍加一份文章列表，仅此而已。
- 打开到看到第一个字：接近零毫秒。

## 快不是性能问题，是伦理问题

每次浪费你 200 毫秒，都是我单方面做的决定——你的时间不如我的设计重要。大多数网站在默认地做这个决定。我不想。

快源自"少"，不是靠 CDN、缓存、代码分割换来的快，而是压根没东西需要加载。静态 HTML、静态 Markdown，走的是 1993 年的老路：服务器交出文件，浏览器展开文件，结束。

## 每个文件是一个宇宙

但我不是原教旨主义者。我有几篇文章是完整的单文件 HTML——带 prototype、带复杂交互、带酣畅淋漓的 CSS。它们躺在 `_html/` 里，不被模板套壳，自给自足。

这遵循一个更古老的原则：**文档应该是完备的**。你把它下载下来，断网，二十年后还能打开。没有外部依赖，没有构建工具，没有过期的 CDN 链接。一个 HTML 文件，就是一个时间胶囊。

简单的 Markdown 承担日常写作，复杂的 HTML 承担一切需要"设计"的表达——两端都是文件，中间没有框架。

## 内容必须独立承重

当你拿掉所有包装——没有精致字体、没有配色、没有动效——剩下的只有文字本身。写得不好，一眼看出来；没有地方可以藏。

这是一种纪律。

## 减法最难

做加法容易：加几个组件、套一个模板、装几个插件，网站"看起来"就专业了。减法难——每减一样，就要承认那东西其实不必要，而默认我们都以为它必要。

1993 年的设计没有过时。是我们发明了很多东西去遮盖一个更简单的真相：**网页本来就是文字。**
